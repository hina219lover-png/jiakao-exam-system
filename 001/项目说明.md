## 🚀 快速开始

### 第一步：下载依赖

用 IDEA 打开项目，右键 `pom.xml` -> `Maven` -> `Reload Project`

### 第二步：初始化数据库

执行 `src/main/resources/db/schema.sql`

### 第三步：配置数据库连接

编辑 `application.yml`：
```yaml
spring:
  datasource:
   username: root
    password: 你的密码
```

### 第四步：补充代码

参考 `下一步操作指南.md` 完成 Service 和 Controller 的实现

### 第五步：启动项目

运行 `DrivingTestApplication.main()`

### 第六步：测试 API

访问：http://localhost:8080/doc.html

## 📊 数据库表结构

### user - 用户表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| username | VARCHAR(50) | 用户名（唯一） |
| password | VARCHAR(100) | 密码（MD5） |
| email | VARCHAR(100) | 邮箱（唯一） |
| phone | VARCHAR(20) | 手机号 |
| role | INT | 角色（0 用户/1 管理员） |
| status | INT | 状态（0 禁用/1 正常） |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |
| deleted | INT | 逻辑删除 |

### question - 题目表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| type | INT | 类型（1 单选/2 判断/3 多选） |
| content | TEXT | 题目内容 |
| options | JSON | 选项（JSON 格式） |
| answer | VARCHAR(10) | 答案 |
| analysis | TEXT | 解析 |
| chapter | VARCHAR(50) | 章节 |
| difficulty | INT | 难度（1-5） |

### exam_record - 考试记录表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| user_id | BIGINT | 用户 ID |
| exam_type | INT | 考试类型 |
| total_score | INT | 总分 |
| score | INT | 得分 |
| passed | INT | 是否及格 |
| start_time | DATETIME | 开始时间 |
| end_time | DATETIME | 结束时间 |
| duration | INT | 用时（秒） |

### answer_record - 答题记录表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| user_id | BIGINT | 用户 ID |
| exam_record_id | BIGINT | 考试记录 ID |
| question_id | BIGINT | 题目 ID |
| user_answer | VARCHAR(10) | 用户答案 |
| correct | INT | 是否正确 |

## 💡 开发建议

1. **先完成 Service 和 Controller** - 这是核心业务逻辑
2. **使用 Knife4j 测试接口** - 比 Postman 更方便
3. **确保后端接口正常** - 再开始前端开发
4. **分阶段开发** - 不要同时修改多个模块
5. **及时提交 Git** - 做好版本管理

## ⚠️ 注意事项

1. **IDEA 中的红色错误** - 右键 pom.xml 重新加载 Maven 依赖即可消失
2. **Lombok 插件** - 确保已安装 Lombok 插件并开启注解处理
3. **Redis 可选** - 如果暂时不安装 Redis，可以先注释掉相关配置
4. **端口占用** - 如果 8080 被占用，修改 application.yml 中的端口号
