## 🚀 快速启动

### 前提条件

1. **JDK 17+** - 确保已安装
2. **Maven 3.6+** - 建议使用 IDEA 内置 Maven
3. **MySQL 8.0+** - 数据库服务
4. **Redis 7.0+** - 缓存服务（可选，用于 Session 存储）

### 第一步：初始化数据库

1. 启动 MySQL 服务
2. 执行数据库脚本：`src/main/resources/db/schema.sql`
3. 或者手动创建数据库和表

```bash
# 方式 1: 使用命令行
mysql -u root -p < src/main/resources/db/schema.sql

# 方式 2: 使用 Navicat/Workbench 等工具打开并执行 schema.sql
```

### 第二步：配置数据库连接

编辑 `src/main/resources/application.yml`，修改数据库配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/driving_test_system?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&createDatabaseIfNotExist=true&useSSL=false
   username: root           # 改成你的 MySQL 用户名
    password: your_password  # 改成你的 MySQL 密码
```

如果需要使用 Redis，也请配置：

```yaml
spring:
  redis:
    host: localhost
    port: 6379
    password: 
```

### 第三步：使用 IDEA 运行项目（推荐）

#### 方法 1: 直接运行主类

1. 用 IDEA 打开项目文件夹 `d:\Study\ideaIU\xiangmu\zonghexiangmu\001`
2. 等待 Maven 自动下载依赖（重要！）
3. 找到文件：`src/main/java/com/driving/DrivingTestApplication.java`
4. 右键 -> Run 'DrivingTestApplication.main()'

#### 方法 2: 使用 Maven 命令

```bash
cd d:\Study\ideaIU\xiangmu\zonghexiangmu\001
mvn clean install -DskipTests
mvn spring-boot:run
```

### 第四步：访问系统

启动成功后，访问：

- **API 文档**: http://localhost:8080/doc.html
- **Swagger UI**: http://localhost:8080/swagger-ui.html

## ⚠️ 常见问题

### Q1: IDEA 中显示很多红色错误（Cannot resolve symbol）？

**解决方法**：
1. 右键点击 `pom.xml`
2. 选择 `Maven` -> `Reload Project`
3. 等待 Maven 下载完所有依赖
4. 如果还有错误，执行：`File` -> `Invalidate Caches` -> `Invalidate and Restart`

### Q2: Lombok 无法识别？

**解决方法**：
1. `File` -> `Settings` -> `Plugins`
2. 搜索并安装 `Lombok` 插件
3. 重启 IDEA
4. 确保开启注解处理：`File` -> `Settings` -> `Build, Execution, Deployment` -> `Compiler` -> `Annotation Processors` -> 勾选 `Enable annotation processing`

### Q3: Redis 未安装怎么办？

**临时方案**：
在 `application.yml` 中注释掉 Redis 相关配置，暂时不使用缓存功能。

**建议**：
安装 Redis for Windows:
```bash
# 使用 Chocolatey 安装
choco install redis-64
```

### Q4: 数据库连接失败？

**检查清单**：
- [ ] MySQL 服务是否启动？
- [ ] 用户名密码是否正确？
- [ ] 数据库 `driving_test_system` 是否已创建？
- [ ] 端口 3306 是否被占用？

### Q5: 端口 8080 被占用？

**解决方法**：
修改 `application.yml`：
```yaml
server:
  port: 8081  # 改成其他端口
```
